Detailed Description

Runtime configuration stored as an INI file on LittleFS.

Hardware constants (pins, I2C addresses) live in include/config.h and are fixed at build time; everything the user may want to tune in the field (station identity, wake interval, LoRa radio parameters) lives here and is loaded from /config.ini at every boot.

If the file is missing (first boot, erased flash) the defaults defined in this class are used and a fresh /config.ini is written back, so the filesystem image upload (pio run -t uploadfs) is optional.

Usage example:

AppConfig config;
config.begin(); // mount FS + load (or create) file
uint64_t us = config.station.wakeIntervalS * 1000000ULL;
